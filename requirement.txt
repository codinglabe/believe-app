Yes — you should add a *Merchant filter* now.

Right now your left filter only shows *Organizations, but the marketplace now has **two seller types*:

* Organizations
* Merchants

So users need a way to narrow products by *seller type* and then by the specific seller.

The cleanest setup is this:

# Best Filter Structure

## 1. Seller Type

Add a new filter block above Organizations:

* All Sellers
* Organizations
* Merchants

When user selects:

* *All Sellers* → show everything
* *Organizations* → show only organization products
* *Merchants* → show only merchant products

---

## 2. Seller List

Below that, the second filter changes dynamically.

### If “Organizations” is selected:

show:

* organization names
* nonprofit names
* causes if needed later

### If “Merchants” is selected:

show:

* merchant names
* merchant stores
* brand names if applicable

### If “All Sellers” is selected:

you can either:

* show both in one combined list, or
* hide seller-specific list until type is chosen

I recommend hiding the detailed seller list until a type is chosen, to keep it clean.

Without this, users cannot easily tell:

* whether they are buying from a nonprofit
* whether they are buying from a merchant

That matters a lot for BIU because part of the value is:

* *shopping merchant products*

# Developer Direction

Tell your developer to add:

## Frontend

Add a new filter group:

* seller_type

Values:

* all
* organization
* merchant

Add seller list filter:

* seller_ids[]

The seller list should refresh based on selected seller type.

---

## Backend logic

Products table or listing query should support:

* seller_type
* organization_id nullable
* merchant_id nullable

Example logic:

* if seller_type = organization → query organization products only
* if seller_type = merchant → query merchant products only
* if seller_ids[] present → filter by those sellers

---

## API / Query Params

Example:

php
/products?seller_type=merchant&seller_ids[]=12&category_ids[]=3


or

php
/products?seller_type=organization&seller_ids[]=4
